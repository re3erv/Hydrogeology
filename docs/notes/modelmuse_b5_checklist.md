# ModelMuse checklist — этап B5 (сравнение RIV / DRN / GHB)

Цель: убедиться, что различия между пакетами `RIV`, `DRN` и `GHB` корректно воспроизводятся на одной и той же сетке и сопоставимых параметрах.

## 1) Открытие проектов

Откройте по очереди три name-файла:

- `models/mf6/05_riv_drn_ghb/riv/mfsim.nam`
- `models/mf6/05_riv_drn_ghb/drn/mfsim.nam`
- `models/mf6/05_riv_drn_ghb/ghb/mfsim.nam`

Проверьте, что каждый проект загружается как MODFLOW 6.

## 2) Общая база моделей

Для всех трёх случаев:

- 1 слой, 10 x 10 ячеек;
- `K = 10` (однородное);
- CHD-границы слева/справа (`10` и `0`);
- пакет сравнения расположен вдоль верхнего ряда (кроме углов с CHD).

## 3) Проверка параметров пакетов

- **RIV**:
  - `stage = 7`;
  - `cond = 20`;
  - `rbot = 5`.
- **DRN**:
  - `elev = 7`;
  - `cond = 20`.
- **GHB**:
  - `bhead = 7`;
  - `cond = 20`.

## 4) Проверка результатов

В корне B5-папки есть:

- `comparison_metrics.csv`
- `comparison_plot.png`
- `run_summary.txt`

Проверьте, что в summary/CSV для каждого пакета есть:

- `center_head`;
- `top_mid_head`;
- `package_exchange_q`;
- `budget_percent_discrepancy`.

## 5) Интерпретация (sanity check)

- `DRN` дренирует только при превышении уровня в ячейке над отметкой дренажа.
- `GHB` допускает двусторонний обмен (в/из модели) в зависимости от разности напоров.
- `RIV` учитывает уровень воды и отметку дна русла (`rbot`), что даёт поведение между «жёсткой» и «ограниченной» связью.
